module和exports的区别和联系

  • 前面的写法中

    1

    exports.x = 10;

  • 和

    1

    module.exports.x = 10;

  • 是一样的效果

  • 但是,对于module.export

  • 你可以这样写

    01.js

    1
2
3
4

    module.exports = {
    x:10,
    y:20
}

  • 然后在导入模块方

    02.js

    1
2
3

    var module_01 = require("./01.js");
console.log(module_01.x);
console.log(module_01.y);

  • 结果正常运行

  • 也就是说,可以将json对象,赋值给module.exports

  • 是否也可以将json对象赋值给exports呢

  • 答案是不可以

  • 会报错

  • 就是说吧

    • module.exports

      既可以是使用json对象的形式

      也可以使用给属性赋值的形式

    • exports

      只能使用给属性赋值的形式

  • 具体为什么

  • 看这个图

  • 对于var的基本数据类型变量

    1
2

    var a = 10;
var b = a;

  • 首先在栈内存(存放key-value)中开辟一块内存空间给变量a,并且存放整型10

  • 然后,var b = a这步

  • 是在栈内存又开辟了一块内存空间给b,并且让b内存空间中的内容和a一样

  • 二者都存放数据10,但这两个10并没什么关系

  • 唯一的关系就是你写的赋值语法

  • 。。。

  • 对于对象

    1
2
3
4
5
6

    var obj1 = {
    x : 10,
    y : 20
}
  
var obj2 = obj1;

  • 都说这叫引用

  • 为什么这么说呢

  • 这是因为

  • 当你创建一个对象变量obj1时,同样会在栈内存开辟一块空间给你的变量obj1

  • 但是你给这个变量obj1赋的值,也就是对象{x:10,y:20}

  • 会被存放在堆内存中

  • 而栈内存中的对象变量obj1,只存放在堆内存中的这个对象{x:10,y:20}的内存地址

  • 这时,你再给

    1

    var obj2 = obj1;

  • 那就是,将obj1里,栈内存里存放的堆内存的对象的内存地址,赋值给obj2

  • 这样,经过一番操作

  • obj1和obj2里的值是一样的

  • 都是堆内存中对象{x:10,y:20}的内存地址

  • 他们俩谁更改对象属性

  • 都是在操作{x:10,y:20}这个对象

  • 现在,言归正传

  • module.exports是初始对象

  • 而exports只是module.exports的引用

    1

    exports = module.exports;

  • 真正有能力将数据暴露给外部的,是module.exports而不是exports

  • 你直接给exports赋值对象,本来exports引用的是module.exports,你现在把exports赋值对象,exports的指向就变了

  • 就没module.exports的能力了

  • 所以说

  • module.exports,你无论是赋值给json对象,还是直接等号赋值

  • 都行

  • 因为module.exports就是老祖宗

  • 从require传递过去的就是module.exports

  • 但你如果更改了exports,给exports赋值一个对象

  • 那它在栈内存中,存储的内存地址就不是module.exports了

  • 所以说,如果你给exports赋值一个对象,那exports就只是一个普通对象了

  • 先说一下计算机的字节原理

  • 在计算机中,最底层数据是二进制

  • 内存中,一个单元格除了能保存0就是1

  • 这里的一个二进制数 - 0或1,代表1位(bit)

  • 所以,位(比特)是计算机底层最小的基本单位

  • 规定,连续的8位 = 1字节(byte)

    8bit = 1 byte / B
    1024byte = 1kb
    1024kb = 1mb
    1024mb = 1gb
    1024gb = 1tb

    一般内存换算是1024进制，也就是2的10次方。1TB=1024GB；1GB=1024MB；1MB=1024KB；1KB=1024Byte。Byte就是B也就是字节；KB是千字节；MB是兆；GB是千兆；TB是千千兆。
KB=1024B；1MB=1024KB=1024×1024B。其中1024=2^10。
1B（byte，字节）= 8 bit（比特，即为位）；
1KB（Kilobyte，千字节）=1024B= 2^10 B；
1MB（Megabyte，兆字节，百万字节，简称“兆”）=1024KB= 2^20 B；
1GB（Gigabyte，吉字节，十亿字节，又称“千兆”）=1024MB= 2^30 B；
1TB（Terabyte，万亿字节，太字节）=1024GB= 2^40 B；
1PB（Petabyte，千万亿字节，拍字节）=1024TB= 2^50 B；
1EB（Exabyte，百亿亿字节，艾字节）=1024PB= 2^60 B；

1BB（Brontobyte，一千亿亿亿字节，珀字节）= 1024YB= 2^90 B；
1NB（NonaByte，一百万亿亿亿字节，诺字节） = 1024 BB = 2^100 B；
1DB（DoggaByte，十亿亿亿亿字节，刀字节） = 1024 NB = 2^110 B；

1XB（Xerobyte，千万亿亿亿亿字节，约字节）=1024CB=2^130 B；[1]
最小的单位是“位”（bit），最大的单位是“约字节”（XB），它们之间相差8.0×2^130倍。
我们在平时生活当中常用“B”“KB”“MB”“GB”和“TB”。其中“MB”和“GB”还有“TB”最为常见。

  • Buffer.allocUnsafe(size)和Buffer.alloc(size)

  • 一样,都是创建一个指定大小的Buffer

  • 唯一不同的是

  • Buffer.alloc(size)

    在给Buffer分配内存空间时,会将待分配内存空间中全部内容清空

    再分配

  • 而Buffer.allocUnsafe(size)

    分配之前不会将待分配这块内存空间中内容清空

    因为内存空间肯定是反复用的

    之前可能有别人用过

    数据还没被清除掉

    所以,这种分配方式

    Buffer中可能带有其他数据

    所以叫作不安全

  • 不安全但是性能好

  • 因为不需要进行清空操作

Step06.文件操作

  • nodejs操作文件需要导入模块fs

  • fs是node的核心模块,无需下载直接require

  • 但是Buffer就不需要require可以直接使用

    1

    var fs = require("fs");

  • 然后就可以通过fs对象来操作文件了

  • node对文件操作分为同步和异步

  • 同步就是程序等待这一块代码执行完才继续向下执行

    • 所谓造成阻塞
    • 同步锁

  • 异步就是,程序执行到这块,不会堵塞,继续向下执行。这块程序有回调函数,通过回调函数获取异步执行结果,确保不会造成异步数据混乱

同步操作

  • 对文件的操作无非三步
    • 1.打开文件
    • 2.对文件进行操作
    • 3.保存并关闭文件
  • 同步操作的方法后面都带有Sync
  • 例如同步打开也是fs.openSync(path)
  • 异步就是fs.open(path,callback)

fs.writeSync(fd,"hello django"); // fs的同步写操作,将第二个参数的内容写进 第一个参数所代表的 文件指针指向的文件中 , 在写之前文件内容会被清空

const http = require("http");

const port = 3000;

// 回调函数,req是请求体,res是响应体
const server = http.createServer((req, res) => {
    res.statusCode = 200; // 响应体状态码
    res.setHeader("Content-Type", "text/plain;charset=utf-8"); // 响应体文本规范(编码等)
    res.write("go"); // write方法用于设置响应内容，可以写多个write函数，参数内容会被按顺序拼接起来
    res.write("python");
    res.end("你好,django!!!"); // 结束响应，end方法中的参数也会作为响应内容，拼接到响应内容最后面
})

server.listen(port, () => { // 监听请求端口
    console.log("This server is running!!!");
})


// 向自己的服务器发起请求
// 第一个参数是 host，第二个是请求参数,第三个是响应体的回调函数
http.get("http://localhost:3000/", "", (res) => {
    let resData = "";
    /*
     不断监听（就是说一个on可以监听所有的write） 服务器中的write方法
     每有一个write方法发送响应内容，on就获取一次最新的write的内容，并将这个内容作为on函数第二个参数，箭头函数的参数
     */
    res.on("data", (data) => {
        resData += data
    });
    res.on("end", () => {
            console.log(resData);
        })
        // console.log(res);
})
  • 就是说，服务器中的所有
  • write方法和end方法
  • 都会被请求脚本的on方法监听
  • 并每次监听都将write或者end写入的响应内容取出
  • 作为on方法第二个参数，也就是箭头函数的参数
  • 实际上就是一个死循环
  • 直到你服务器执行end方法
  • 否则on方法一直执行下去
