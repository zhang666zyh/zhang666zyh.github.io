Step01.Unity中导入EasyAR资源并运行第一个项目

导入SDK

  • 注册账号

  • 下载Unity的EasyAR插件

    • 点击下载

    • 点击Unity Plugin

    • 下载zip包

    • 新版本下载的zip包解压后是什么TGZ文件

    • 关于新版本的导入官方文档是这样说的

      Windows->PackageManager->点击左上角加号->Add package from tarball->选择解压出的TGZ文件

    • 导入后,tgz文件在哪个目录下也不要删了,项目就找这个文件.删除会导致项目无法启动

填写许可证

  • 到官网去注册一个License key作为项目支持密钥

  • 注册后找到开发中心

  • 点击我需要新的许可密钥

  • 选择个人版

  • 正常填写

    • 如果要导出Android应用

    • Package Name需要和Unity的相同

      com.公司名.项目名

  • 确认后点开生成的密钥

  • 复制

  • 导入TGZ文件后,Unity导航栏就会出现一个EasyAR标识

    EasyAR->Sense->configuration

  • 找到右侧Inspector最下面的EasyAR SDK License Key

  • 将上面复制的密钥粘贴进去

  • 至此,EasyAR的Unity环境就配置好了

第一个项目(图像跟踪)

  • 这第一个项目就用图像跟踪举例,因为官方文档前面说了运动跟踪等情况但没实现

  • EasyAR无需像ARFoundation这样删除AR Camera

  • 点击Main Camera,进行配置

  • 官网是这样讲的

    • Tag: 如果Camera不是来自 AR Foundation 或 AR Engine Unity SDK 或 Nreal SDK ，可以设置Camera Tag为 MainCamera，这样它会在AR Session启动时被frame source所选用。或者，你也可以通过在Inspector面包设置 [FrameSource.Camera](https://help.easyar.cn/EasyAR Sense Unity Plugin/latest/ApiReference/FrameSource.html#camera) 来修改 FrameSource 的 Camera 为这个Camera。
    • Clear Flags: 需要选择为 Solid Color 以确保Camera图像可以正常渲染。如果选择为 Skybox ，Camera图像将无法显示。
    • Background: 这个非必需配置，考虑到使用体验，建议将背景颜色设为黑色以便在Camera设备打开前和切换时以黑色显示。
    • Clipping Planes: 根据识别物体实际的物理距离设置。这里设置Near为0.1（米）以避免相机离物体较近时无法显示。

创建 EasyAR AR Session

  • 比如，如果你要使用 图像跟踪，你可以使用 EasyAR Sense > Image Tracking > AR Session (Image Tracking Preset) 来创建AR Session
  • 其他情况,例如稀疏地图、运动跟踪等官方文档有
  • 有了AR Session就有了AR Camera,接下来创建Image Target

创建Image Target

  • 如果你要使用 图像跟踪 ，你需要通过 EasyAR Sense > Image Tracking > Target : Image Target 创建 [ImageTargetController](https://help.easyar.cn/EasyAR Sense Unity Plugin/latest/ApiReference/ImageTargetController.html)

  • 这时候场景中ImageTarget应该会显示成问号

  • 然后需要配置这个 ImageTarget。这通常由好几种不同的方法。这里使用其中的一种，使用StreamingAssets中的图片创建target。

  • 在Assets中创建StreamingAssets文件夹

  • 将需要识别的图片拖入StreamingAssets中。这里选用名片图。

  • 然后配置这个 [ImageTargetController](https://help.easyar.cn/EasyAR Sense Unity Plugin/latest/ApiReference/ImageTargetController.html) 以使用 StreamingAssets 中的图片

  • 不像Vuforia还要创建数据库导包

  • 这是官网对于Image Target配置的描述

    • Source Type: 这里设置为 Image File ，表示将使用图片文件创建ImageTarget。
    • Path Type: 这里设置为 StreamingAssets ，表示 Path 将使用相对于StreamingAssets的路径。
    • Path: 图片相对于StreamingAssets的路径(就是要识别追踪的图片在StreamingAssets目录下的路径)
    • Name: target名字，可以随意输入。
    • Scale: 根据识别图宽度实际的物理大小设置。这里使用的名片实际大小为9mm，因此设为0.09（米）。
    • Tracker: 需要加载 [ImageTargetController](https://help.easyar.cn/EasyAR Sense Unity Plugin/latest/ApiReference/ImageTargetController.html) 的 [ImageTrackerFrameFilter](https://help.easyar.cn/EasyAR Sense Unity Plugin/latest/ApiReference/ImageTrackerFrameFilter.html) 。在添加ImageTarget时，它会被默认设为场景中的其中一个 [ImageTrackerFrameFilter](https://help.easyar.cn/EasyAR Sense Unity Plugin/latest/ApiReference/ImageTrackerFrameFilter.html) ，添加后仍可修改。

  • 场景中ImageTarget的显示将随Path的输入而随时改变

  • 在ImageTarget下创建一个任意的3D物体

  • 运行项目

Step02.识别多张图片

  • 创建两个Image Target,放上StreamingAssets目录下不同图片,创建两个不同的3D物体

  • 更改AR Session下的Image Tracker的属性值

    Simultaneous Target代表最大识别图片数

    默认是1,改成2即可

  • 但这里有问题

  • 当放置两张一样图片识别的时候,它只在其中一个上面显示3D物体

    public ImageTargetController targetController;

    {
        if (targetController)
        {
            targetController.TargetFound += () =>
            {
                Debug.LogFormat(...);	//当图像被跟踪到
            };

            {
                Debug.LogFormat(...);	//当图像从视野消失
            };
            targetController.TargetLoad += (Target target, bool status) =>
            {
                Debug.LogFormat(...);	//加载图像
            };
            targetController.TargetUnload += (Target target, bool status) =>
            {
                Debug.LogFormat(...);	//卸载图像
            };
        }
    }
    public void SetTracker(bool status)
    {
        tracker.enabled = status;	//设置是否跟踪
    }
    public void SetTarget(bool status)
    {
        if (status)
        {
            targetController.Tracker = tracker; //加载图像
        }
        else
        {
            targetController.Tracker = null;    //卸载图像
        }
    }

Step03.播放视频

在图片上播放视频

  • 在场景中设置Main Camera的Clear Flags属性为Solid Color
  • 创建Image Target,并设置识别图片
  • 选中一个“ImageTarget”，点击鼠标右键，在弹出菜单中选择3D Object–Plane,为“ImageTarget”对象添加一个平面作为其子游戏对象
  • 设置Plane游戏对象的角度为90,0,180，并修改其Scale属性将其大小设置的和视频图片大小一样(就是横长竖窄那种(0.1,0,0.05))
  • 选中Plane游戏对象，点击菜单Add Component–Video–Video Player，为其添加一个视频播放组件
  • 将一段视频拖到目录Videos下，导入为资源，并将其拖动到Video Player组件的Video Clip属性中为其赋值
  • 运行项目

在其它3D物体上播放视频

  • 就是将上面的平面替换成立方体等,然后给3D物体一个适当的角度

  • 别整个平面看不出来是3D的

  • 比如立方体设置(45,45,0),它的一个角就扎在图片上了

  • 运行项目,立方体6个面都播放同一个视频

    using System.Collections;
using System.Collections.Generic;
using UnityEngine;
  
public class ChangeMaterial : MonoBehaviour
{
    public Material Blue;
    public Material K;
    private bool isClick = false;
    // Use this for initialization
    void Start () {    
    }
    // Update is called once per frame
    void Update () {   
    }
    void OnMouseDown()
    {
        if (!isClick) {
            this.gameObject.GetComponent<MeshRenderer> ().material = Blue;
            isClick = true;
        } else {
            this.gameObject.GetComponent<MeshRenderer> ().material = K;
            isClick = false;
        }
    }
}

    using UnityEngine;
using System.Collections;
public class Enlarge : MonoBehaviour {
    private Touch oldTouch1;  //上次触摸点1(手指1)
    private Touch oldTouch2;  //上次触摸点2(手指2)
    void Update()
    {
        //没有触摸，就是触摸点为0
        if (Input.touchCount <= 0)
        {
            return;
        }     
        //多点触摸, 放大缩小
        Touch newTouch1 = Input.GetTouch(0);
        Touch newTouch2 = Input.GetTouch(1);
        //第2点刚开始接触屏幕, 只记录，不做处理
        if (newTouch2.phase == TouchPhase.Began)
        {
            oldTouch2 = newTouch2;
            oldTouch1 = newTouch1;
            return;
        }
        //计算老的两点距离和新的两点间距离，变大要放大模型，变小要缩放模型
        float oldDistance = Vector2.Distance(oldTouch1.position, oldTouch2.position);
        float newDistance = Vector2.Distance(newTouch1.position, newTouch2.position);
        //两个距离之差，为正表示放大手势， 为负表示缩小手势
        float offset = newDistance - oldDistance;
        //放大因子， 一个像素按 0.01倍来算(100可调整)
        float scaleFactor = offset / 100f;
        Vector3 localScale = transform.localScale;
        Vector3 scale = new Vector3(localScale.x + scaleFactor,
            localScale.y + scaleFactor,
            localScale.z + scaleFactor);
        //在什么情况下进行缩放
        if (scale.x >= 0.05f && scale.y >=0.05f && scale.z >= 0.05f)
        {
            transform.localScale = scale;
        }
        //记住最新的触摸点，下次使用
        oldTouch1 = newTouch1;
        oldTouch2 = newTouch2;
    }
}

    using System.Collections;
using System.Collections.Generic;
using UnityEngine;
  
public class Drag : MonoBehaviour {
    private Vector3 _vec3TargetScreenSpace;// 目标物体的屏幕空间坐标
  
    private Vector3 _vec3TargetWorldSpace;// 目标物体的世界空间坐标
  
    private Transform _trans;// 目标物体的空间变换组件
  
    private Vector3 _vec3MouseScreenSpace;// 鼠标的屏幕空间坐标
  
    private Vector3 _vec3Offset;// 偏移
  
    void Awake() { _trans = transform; }
  
    IEnumerator OnMouseDown()
  
    {
  
        // 把目标物体的世界空间坐标转换到它自身的屏幕空间坐标
  
        _vec3TargetScreenSpace = Camera.main.WorldToScreenPoint(_trans.position);
  
        // 存储鼠标的屏幕空间坐标（Z值使用目标物体的屏幕空间坐标）
  
        _vec3MouseScreenSpace = new Vector3(Input.mousePosition.x, Input.mousePosition.y, _vec3TargetScreenSpace.z);
  
        // 计算目标物体与鼠标物体在世界空间中的偏移量
  
        _vec3Offset = _trans.position - Camera.main.ScreenToWorldPoint(_vec3MouseScreenSpace);
  
        // 鼠标左键按下
  
        while (Input.GetMouseButton(0))
  
        {
  
            // 存储鼠标的屏幕空间坐标（Z值使用目标物体的屏幕空间坐标）
  
            _vec3MouseScreenSpace = new Vector3(Input.mousePosition.x, Input.mousePosition.y, _vec3TargetScreenSpace.z);
  
            // 把鼠标的屏幕空间坐标转换到世界空间坐标（Z值使用目标物体的屏幕空间坐标），加上偏移量，以此作为目标物体的世界空间坐标
  
            _vec3TargetWorldSpace = Camera.main.ScreenToWorldPoint(_vec3MouseScreenSpace) + _vec3Offset;
  
            // 更新目标物体的世界空间坐标
            _trans.position = _vec3TargetWorldSpace;
            // 等待固定更新
            yield return new WaitForFixedUpdate();
  
        }
  
    }
}
