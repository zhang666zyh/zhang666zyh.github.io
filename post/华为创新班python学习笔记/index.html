Step06.List、Set、Turple和Dict的区别

  • List

    列表,可变序列,用[]括起来,可进行CURD操作,可以通过索引访问元素,且索引可以为负数

  • Turple

    元组,不可变序列,用()括起来,只能进行访问操作,访问方式和List一样

  • Set

    集合,可变序列,不同于List和Turple,不能通过索引访问,且集合中元素无序,且可以自动去重

    # -*- encoding:UTF-8 -*-
  
"""
@File     :   re_003.py
@Time     :   2021/9/26 7:19
@Author   :   张宇航
@Contact  :   zhangzyh666@163.com
@MyWeb    :   https://zhang666zyh.github.io
"""
import re
  
pattern = '(.*),(.*)'
string = 'Hello,World'
  
result = re.match(pattern,string)
if result:
    print("匹配成功,所有结果为:" + result.group())
    print("第一组结果为:" + result.group(1))
    print("第二组结果为:" + result.group(2))
else:
    print("匹配失败!!!")

  • 关于match方法的flag参数

    • flag是指定匹配格式,例如

      • re.i:区分大小写

      • re.M:支持换行匹配

      • re.S:匹配内容包括换行符\n

        如果没有re.S,则只在每一行进行匹配,每行单独为一个整体

        re.S则为,在意\n符,将换行符上下文作为一个整体

  • 常用正则表达式总结:

    符号 解释
    ^string pattern只能出现在string开头
    string$ pattern只能出现在string结尾
    ? 出现0次或1次
    + 出现1次或多次
    * 出现0次或多次
    . 匹配任意字符
    {2} 字符出现2次
    {2,} 字符至少出现2次
    {2,4} 字符出现2-4次
    a|b 或,二选一
    [a,b,c] 多选一
    [^a,b,c] 除了a,b,c之外的所有字符
    [a-z] 所有的小写字母
    [A-Z] 所有的大写字母
    \w 匹配数字、字母、下划线
    \W 匹配除数字、字母、下划线以外的所有字符
    \s 匹配空字符,空格、换行符、制表符
    \S 匹配除空字符,空格、换行符、制表符以外的所有字符
    \d 匹配0-9所有的数字
    \D 匹配所有除数字以外的所有字符
    \1、\2、\3… 匹配第n个圆括号中的内容
    \ 转义字符

def run1(lock):
    for i in range(1, 101):
        lock.acquire()
        print("子线程1->%d" % i)
        lock.release()


def run2(lock):
    for i in range(1, 101):
        lock.acquire()
        print("子线程2->%d" % i)
        lock.release()


def main():
    for i in range(1, 41):
        if i <= 20:
            print("这是主进程->%d" % i)
            if i == 20:
                p1 = multiprocessing.Process(target=run1, args=(lock,))
                p1.start()
                p1.join()
        else:
            print("这是主进程->%d" % i)
    p2 = multiprocessing.Process(target=run2, args=(lock,))
    p2.start()


if __name__ == '__main__':
    main()

哲学家就餐问题可以这样表述，假设有五位哲学家围坐在一张圆形餐桌旁，做以下两件事情之一：吃饭，或者思考。吃东西的时候，他们就停止思考，思考的时候也停止吃东西。餐桌中间有一大碗意大利面，每位哲学家之间各有一只餐叉。因为用一只餐叉很难吃到意大利面，所以假设哲学家必须用两只餐叉吃东西。他们只能使用自己左右手边的那两只餐叉。哲学家就餐问题有时也用米饭和五根筷子而不是意大利面和餐叉来描述，因为吃米饭必须用两根筷子。

这个问题不考虑意大利面有多少，也不考虑哲学家的胃有多大。假设两者都是无限大。

问题在于如何设计一套规则，使得在哲学家们在完全不交谈，也就是无法知道其他人可能在什么时候要吃饭或者思考的情况下，可以在这两种状态下永远交替下去。

import threading
import time

lock1 = threading.RLock()
lock2 = threading.RLock()
lock3 = threading.RLock()
lock4 = threading.RLock()
lock5 = threading.RLock()


class Philosopher:
    def __init__(self, left, right):
        self.left = left
        self.right = right


philosopher1 = Philosopher(lock5, lock1)
philosopher2 = Philosopher(lock1, lock2)
philosopher3 = Philosopher(lock2, lock3)
philosopher4 = Philosopher(lock3, lock4)
philosopher5 = Philosopher(lock4, lock5)


def run(philosopher, name):
    lockLeft = philosopher.left.acquire()
    if lockLeft:
        print(name, "获取左筷子")

    lockRight = philosopher.right.acquire()
    if lockRight:
        print(name, "获取右筷子")
        print("哲学家", name, "开始就餐")
        time.sleep(1)

    philosopher.right.release()
    philosopher.left.release()


t1 = threading.Thread(target=run, args=(philosopher1, "philosopher1"))
t2 = threading.Thread(target=run, args=(philosopher2, "philosopher2"))
t3 = threading.Thread(target=run, args=(philosopher3, "philosopher3"))
t4 = threading.Thread(target=run, args=(philosopher4, "philosopher4"))
t5 = threading.Thread(target=run, args=(philosopher5, "philosopher5"))

t1.start()
t2.start()
t3.start()
t4.start()
t5.start()

  • Python中Lock与RLock的区别

    在同一线程内,对RLock进行多次acquire()操作,程序不会阻塞.因为它是递归锁

    而Lock多次acquire会产生死锁

    RLock进行了多少次acquire,实际上就需要多少个release来释放
