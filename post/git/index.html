Step01.开发一个项目流程

  • 创建一个仓库,默认分支是master

  • 在终端,将码云上的所有分支拉到本地

    1
2

    git pull
# 输入用户名和密码

  • 开发项目,当前项目默认分支是master,所以你开发的所有代码都在,本地的master分支

  • 提交作业

    1
2
3
4

    git add . # 将当前项目中文件,添加到本地当前分支中
git commit -m 'xxx'
git push
# 输入用户名和密码

  • 但是,一个项目不可能只有一个分支

  • 如果是有多个模块,可能需要创建多个分支

  • 在码云创建分支

  • 创建的分支,里面的代码是从默认分支,也就是master中复制过来的

  • 然后将码云上的所有分支拉到本地

    1
2

    git pull
# 输入用户名和密码

  • 切换到某个分支

    1

    git checkout 分支名

  • 在该分支下开发代码

  • 提交作业

    1
2
3
4

    git add . # 将当前项目中文件,添加到本地当前分支中
git commit -m 'xxx'
git push
# 输入用户名和密码

  • 但是,master中也需要提交,和刚才新创建的分支中相同的代码

  • 不需要add .、commit啥的,直接将刚才新建的分支合并到默认分支

    1

    git merge origin/分支名

  • 这样,就将某个分支合并到了当前分支

Step04.GitHub + hugo搭建个人博客

Windows10配置hugo+github个人博客

配置hugo环境

下载hugo

  • 安装hugo压缩包
  • 随便一个地址新建一个文件夹存放hugo环境
  • 例如D://hugo
  • 在hugo下创建两个文件夹bin和site
  • 将下载的压缩包中的三个文件解压到bin目录中

配置环境变量

  • 将bin目录分别添加到环境变量上下栏的Path中

下载模板

  • 下载地址

  • 创建一个hugo网站,然后将下载的模板配置进去

    一般是将模板中的theme目录中的/themes/./exampleSite下的文件和/themes下的config.toml文件复制到网站主目录中用于替换主目录中空的同名文件夹

  • 配置好后启动

    1

    hugo server

推送到GitHub

  • 在GitHub上创建一个Repository(仓库)，命名为:GitHub用户名.github.io -> 比如我的Repo就叫zhang666zyh.github.io

  • 为了命名规范,新建站点的时候执行hugo new site zhang666zyh,站点名和GitHub用户名一致

  • 然后在你的Repo里面点击Settings, 下拉，找到Github Pages

  • 进入博客文件的根目录,执行以下代码

    1
2
3
4
5
6
7

    hugo (生成public文件夹)
cd public
git init
git add .
git commit -m "comments"
git remote add origin https://github.com/zhang666zyh/zhang666zyh.github.io.git
git push -f --set-upstream origin master

    第五步这里可能会报错

  • 最后一步push它会弹出一个窗口,到浏览器登陆GitHub账号,然后就可以push了

  • 如果报错为Could not github.com之类,那就打开C:\Windows\system32\drivers\etc\hosts文件

Github在本机git生成SSH公钥

1.通过命令窗口

a. 打开你的 git bash 窗口

b. 进入 .ssh 目录：cd ~/.ssh

c. 找到 id_rsa.pub 文件：ls

d. 查看公钥：cat id_rsa.pub 或者 vim id_rsa.pub

如图：

2.或者你也可以直接输入命令 ：cat ~/.ssh/id_rsa.pub

3.或者你也可以直接打开你用户（一般都是 Administrator）下的 .ssh 文件夹，打开它里面的 id_rsa.pub 文件，如图：

何谓公钥

  • 1.很多服务器都是需要认证的，ssh认证是其中的一种。在客户端生成公钥，把生成的公钥添加到服务器，你以后连接服务器就不用每次都输入用户名和密码了。
  • 2.很多git服务器都是用ssh认证方式，你需要把你生成的公钥发送给代码仓库管理员，让他给你添加到服务器上，你就可以通过ssh自由地拉取和提交代码了。

生成公钥

  • 1.如果通过上面的方式找不到公钥，你就需要先生成公钥了：ssh-keygen
  • 2.接着会确认存放公钥的地址，默认就是上面说的路径，直接enter键确认
  • 3.接着会要求输入密码和确认密码，如果不想设置密码直接不输入内容 按enter键
